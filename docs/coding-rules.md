# Coding Rules & Best Practices

## General
- All code must be production-ready
- Use env variables for all secrets/IDs
- Write clean, DRY, and modular code
- Document all functions/components
- Use TypeScript (frontend)
- Use Python 3.11+ (backend)
- Validate all user input
- Use Docker for local/dev
- All endpoints/components must be tested

## Frontend
- Use Next.js, React, Tailwind
- All UI must be responsive
- Use Headless UI for modals/drawers
- Use Zustand for state
- Use React Hook Form for forms
- Defer all pixel scripts
- Deduplicate events
- Use Framer Motion for animation

## Backend
- Use FastAPI, SQLAlchemy, Pydantic
- Use Alembic for migrations
- Hash PII for CAPI
- Deduplicate events
- Log all events
- Use OpenAPI docs
